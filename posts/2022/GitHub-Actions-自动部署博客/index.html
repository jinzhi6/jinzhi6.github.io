GitHub Actions 自动部署博客

太菜了，天天踩坑！不懂得东西，需要脚踏实地去学习！吃一堑【qian】，长一智！

部署到Github

创建仓库

  • 公有仓库：
    • jinzhi-liu.github.io：在Github装博客的静态网页
    • jinzhi：在Gitee装博客的静态网页
  • 私有仓库
    • blog：在Github装博客的源代码

本机保存了私钥1【保证本机可以上传文件到blog私有仓库】

在blog仓库添加公钥1【保证本机可以上传文件到blog私有仓库】

在blog仓库添加私钥2【保证blog仓库可以上传文件到jinzhi-liu.github.io公有仓库】

打开Actions secrets，新建密钥，填入 C:\Users\lim.ssh\hexo-deploy-key 的内容

Repository secrets名称为HEXO_DEPLOY_KEY

在jinzhi-github.io仓库添加公钥2【保证blog仓库可以上传文件到jinzhi-liu.github.io公有仓库】

打开Deploy keys，新建公钥，填入C:\Users\lim\.ssh\hexo-deploy-key.pub的内容

Title为HEXO_DEPLOY_PUB，并勾选Allow write access

创建工作流 Workflow【配置一定要对】

在根目录新建根目录\.github\workflows 2个文件夹，并新建deploy.yml

简单解释一下，当我们推送内容到远程 master 分支的时候，就会触发这个 Workflow。添加以下配置

本地Push博客源代码到blog私有仓库

删除[Blogroot]/themes/butterfly/.git，存在.git就会识别为子项目，从而无法上传

在根目录运行

添加屏蔽项，新建[Blogroot]/.gitignore

再运行 git 提交指令，将博客源码提交到 github 上

出现错误 ! [rejected] main -> main (fetch first) error: failed to push some refs to 'git@github.com:jinzhi-liu/jinzhi-liu.git' hint: Updates were rejected because the remote contains work that you do

【解决方案】：GitHub 远程仓库中的 README.md 文件不在本地仓库中。

Git 提交时出现错误：ERROR: Repository not found. fatal: Could not read from remote repository.

【原因】没有在github配置公钥

同步到Gitee

在Gitee的个人设置里面添加SSH公钥2【保证jinzhi-github.io仓库可以上传文件到jinzhi公有仓库】
